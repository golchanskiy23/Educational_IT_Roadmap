# Особенности пакета unsafe

Существует пакет, с помощью которого можно проникать внутрь безопасной среды Go - пакет unsafe.

## Он позволяет
- Конвертировать типы указателей
- Достучаться до приватных полей структур
- Лезть напрямую в память и менять данные
- Работа с выравниванием данных

Плата - код может стать крайне нестабильным + изменение философии языка Golang как безопасного языка.

## Unsafe pointer
Он не знает, что находится за типом данных по адресу, и его можно использовать для конвертации указателей.
```go
var p unsafe.Pointer
```

- Важно, что подрубается адресная арифметика, то есть преобразовав в сырой указатель, можем двигать адрес побитово. Но это черевато попаданием в невалидный адрес.

## Основные функции пакета
- unsafe.Sizeof: возвращает размер объекта в байтах
- unsafe.Alignof: возвращает выравнивание объекта
- unsafe.Offsetof: возвращает смещение поля структуры

Выравнивание — это требование процессора, чтобы определённые типы данных начинались по адресам, кратным некоторому числу байт.

Выравнивание нужно для эффективного доступа к памяти:
Чтение/запись по выровненным адресам быстрее.
На некоторых архитектурах доступ к невыравненным данным может вызвать паники или ошибки.

## Взаимосвязь выравнивания и смещения
- Выравнивание показывает какому адресу должен быть кратен адрес переменной

- Смещение - реальная позиция в памяти

- Размер структуры должен быть кратен максимальному выравниванию её полей

```go
// Пример 1
type S1 struct {
	A int8
	B int32
	C int16
}
// A (align 1) -> offset(0)
// B (align 4) -> padding(3 байта), чтобы могли поставить 4-байтовое число -> offset(4)
// C (align 2) -> offset(8)
// 1+3+4+2+2 = 12 байт

// Пример 2
type S2 struct {
	A int8 // align(1), offset(0)
	B int64 // align(8) , offset(8)
	C int8 // align(1), offset(16) -> 24 байта
}

// Пример 3
type S3 struct {
	A int64 // align(8), offset(0)
	B int8 // align(1) , offset(8)
	C int8 // align(1), offset(9) -> 16 байта
}
```

## Риски использования unsafe
- Код становится хрупким. Любое изменение структуры данных может всё сломать.
- Отсутствие гарантий от Golang.
- Сложная отладка. Ошибки при работе с unsafe не всегда легко обнаружить.
