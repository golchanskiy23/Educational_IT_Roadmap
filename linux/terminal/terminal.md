# Linux CLI

## Навигация и файлы

### ls
ls [опции] [директория]

Выдача информации о файлах или каталогах. Фвйлы -> каталоги в алфавитном порядке
- ls -a - вывод всех файлов, в том числе скрытых
- ls -l - вывод в длинном формате, включая режим доступа, владельца, группу, время модификации, размер файлов в байтах, число ссылок на файл

```bash
ls -l /util/by
# ugo - тип файла, user, group, other, итого 9 бит на доступ
-rwxr-xr-x   1 root     sys           50 Jun 22 10:42 /util/by
```

r - чтение, w - запись, x - запуск, '-' - отсутствие доступа

### find
find [директория] [опции] [действие] 

Поиск файлов по любому параметру: имя, содержимое, дата добавления и т.д. Шаг за шагом проходит через всю иерархию директорий.

```bash
$ find . -name ".txt"
./s.txt

$ find . -iname "s.txt"
./s.txt
./S.txt

$ find . -name "*.png"
./babutafb.png
./babutafacebook.png
./Moodle2.png
./moodle.png
./moodle/moodle1.png
./genxfacebook.png

```

- 2>/dev/null - перенаправляет сообшения об ошибках по сути в никуда

Поиск по типу файла:
- f – простые файлы;
- d – каталоги;
- l – символические ссылки;
- b – блочные устройства (dev);
- c – символьные устройства (dev);
- p – именованные каналы;
- s – сокеты;

```bash
# по размеру
$ find . -size +1G
./Microsoft_Office_16.29.19090802_Installer.pkg
./android-studio-ide-183.5692245-mac.dmg
```

```bash
# atime - поиск файлов по времени последнего доступа 
# здесь ищет файлы к которым не обращались последние 180 дней
$ find . -atime +180
```

К команде find можно добавить действия, которые будут произведены с результатами поиска:
- -delete — Удаляет соответствующие результатам поиска файлы
- -ls — Вывод более подробных результатов поиска с: размерами файлов, количеством inode.
- -print Стоит по умолчанию, если не указать другое действие. Показывает полный путь к найденным файлам.
- -exec Выполняет указанную команду в каждой строке результатов поиска.

```bash
# поисковые параметры можно соединять и выводить
$ find . -type f -and -user bloom -and -size +500M -and -atime +10 -exec ls -lah {} \;
```

### locate
locate [опции] [шаблон_для_поиска]

Используется для поиска файлов, расположенных на машине пользователя или на сервере, однако ведёт поиск в собственной базе данных, обновляемой через cron командой updatedb. Но из-за нечастого обновления данные могут быть неактуальными. Более скудный набор фильров (только по имени и пути).

```bash
$ locate filename        # найти файл по имени
$ locate -i filename     # без учёта регистра
$ locate -c filename     # только количество совпадений
$ locate -n 5 filename   # показать первые 5 результатов
$ locate -r '\.mkv$'     # поиск по регулярному выражению
$ sudo updatedb          # обновить базу данных вручную
```

### tree
tree [опции]

Изображает структуру каталогов в виде дерева, причём имеет широкую почву для фильтрации параметров и форматирования вывода.

```bash
# выводит только директории(d), включая скрытые(a) и время их последнего изменения(D), v(упорядочивание по вложенности директорий) + существует ограничение на число файлов, причём меньше 25 не установить
$ tree -avdD --filelimit 26
```

Опять же куча параметров -> в man.

## Работа с текстом

### cat
cat [опции] [file1] [file2] [...] 

Используется для чтения, создания и объединения файлов.

```bash
# стандартный поток ввода обозначается как '-', т.о. комбирируем ввод и вывод и нумеруем все строки(n)
$ cat -n file1 - file2 
# нумеруем только непустые строки(b)
$ cat -b file 
```

При отсутствии переданных файлов cat будет читать данные из стандартного ввода(на экран). Также можно записывать данные из стандартного потока в файл.
```bash
# здесь выводим содердимое двух файлов и записываем в третий
$ cat file1 file2 > file3
```

### less
less [опции] [файл]

Для просмотра содержимого файла и вывода небольшими фрагментами. Зачастую используется для просмотра больших файлов и листать в обоих направлениях. 

При открытии файла открывается командная строка, в которой можно искать вниз/вверх по файлу -> (/error) / (?error). Также там работают регулярки.

#### Некоторые команды:
```bash
# нумерация строк файла
$ less -N /var/log/syslog
# открытие файла с текущей позиции
$ less +100 bigfile.txt
# ВАЖНО: переход в режим реального времени - новые строки появляются автоматически
$ less +F /var/log/nginx/access.log
```

#### Для нескольких файлов:
```bash
# с помощью командной строки переключаться между файлами
# :n - следующий, :p - предыдущий, :d - удаление текущего из списка
$ less file1.log file2.log file3.log
```

#### Пайпы
less также позволяет выводить вывод других команд через конвейер

```bash
$ dmesg | less
```

#### Сравнение инструментов

| Инструмент | Прокрутка назад | Поиск         | Загрузка файла | Слежение |
|------------|-----------------|---------------|----------------|----------|
| `cat`      | —               | —             | Целиком        | —        |
| `more`     | —               | Только вниз   | Целиком        | —        |
| `less`     | Да              | В обе стороны | Кусками        | `+F`     |

#### Шпаргалка less

| Задача                  | Команда / клавиша  |
|-------------------------|--------------------|
| Открыть файл            | `less filename`    |
| Открыть с позиции конца | `less +G filename` |
| Слежение за файлом      | `less +F filename` |
| Номера строк            | `less -N filename` |
| Сжать пустые строки     | `less -s filename` |
| Игнорировать регистр    | `less -i filename` |
| Поиск вниз              | `/pattern`         |
| Поиск вверх             | `?pattern`         |
| Следующее совпадение    | `n`                |
| Предыдущее совпадение   | `N`                |
| В начало файла          | `g`                |
| В конец файла           | `G`                |
| Следующий файл          | `:n`               |
| Выйти                   | `q`                |


### head
head [опции] [файл1] [файл2] [...]

Вывод начальных строк(по умолчанию 10) для каждого файла.

```bash
# вывод первых 15 строк
$ head -n 15 file1.txt
# вывод первых 500 байт
$ head -c 500 file1.txt
```

- аналогично для tail

### tail -f
Выполняет ту же функцию, что и less +F, то есть следит за файлами в реальном времени.

```bash
# начать с последних 50 строк и следить
$ tail -n 50 -f /var/log/nginx/access.log
# следить за несколькими файлами сразу
$ tail -f /var/log/nginx/access.log /var/log/nginx/auth.log
```

Из минусов - отсутствие переключения между выводом файла и режимом слежки (в less это доступно).

### sort
sort [опции] [file1] [file2] [...]

Утилита для вывода текстовых строк в определенном порядке. Проще говоря, для сортировки. Опции определяют критерий сортировки.

```bash
# сортировка по 2-ой колонке по числам
$ ls -l | sort -nk2
# можно также задавать приоритеты сортировки
$ ls -l | sort -nk5,5 -k9
```

### uniq
uniq [опции] [файл для чтения] [файл для записи]


Позволяет работать с повторяющимися строками, допустим выводить только их или удалить их из файла.
```bash
# сравнение по первым пяти символам
$ echo -e небо исполосовано молниями\\nоблака на небе\\nоблака разогнал ветер\\nоблака закрыли солнце\\nсолнце светит ярко\\nзвезды кажутся огромными | uniq -w5
# убирает из вывода все дубликаты
$ echo -e небо\\nоблака\\nоблака\\nоблака\\nсолнце\\nзвезды | uniq
# выводит только уникальные строки
$ echo -e небо\\nоблака\\nоблака\\nоблака\\nсолнце\\nзвезды | uniq -u
```

### cut
cut [опция] [файл]

Используется для вырезания части текста.
Можно вырезать как символы, так и байты, причём как отдельно-взятые, так и диапазон.

```bash
# также можно вырезать первые элементы по столбцам. По разделителю ':'
$ cut -d ':' -f 1 cut_command.txt
# здесь результат - первые 3 слова
$ echo "He was extraordinarily particular about politeness in others." | cut -d ' ' -f 1,2,3
# по аналогии с срезами go можем выделять диапазоны вырезаемых элементов
# здесь первые 2
$ echo "We became still more friendly." | cut -d ' ' -f -2
```

## Regex

### Термины

#### Регулярные выражения

Регулярные выражения — это строка символов, которые выражает шаблон поиска, часто сокращается
как Regex или Regexp. Regexp используется для поиска или замены слов в текстах, кроме того,
мы можем проверить, соответствует ли текст установленным нами правилам или нет.

**Пример:** у вас есть список имен файлов и вы хотите найти только файлы с расширением pdf.
После ввода выражения `^\w+\.pdf$` это будет работать.

#### Базовые сходства

Символ или слово, которое мы хотим найти, написано напрямую — это похоже на обычный процесс поиска.

```bash
echo "bar ber bir bor bur" | rg "bar"
```

##### Любые символы

Точка `.` позволяет выбрать любой символ, включая специальные символы и пробелы.

#### Наборы символов

Если одним из символов в слове могут быть разные символы, мы записываем его в квадратные
скобки `[]` со всеми альтернативными символами.

```bash
echo "bar ber bir bor bur" | rg "b[aeiou]r"
```

##### Наборы негативных символов

Если нужны исключения, то перед символами ставится символ возврата каретки `^`, например, `[^abc]`.

```bash
echo "bar ber bir bor bur" | rg "b[^ae]r"
```

##### Буквенный диапазон

Чтобы найти буквы в указанном диапазоне, начальная буква и конечная буква заключаются в квадратные
скобки `[]` с тире между ними `-`. Регистр учитывается.

```bash
echo "abcdefghijklmnopqrstuvwxyz" | rg "[a-w]"
```

##### Цифровой диапазон

То же самое — начальный номер и конечный номер записываются в квадратных скобках `[]` с тире `-` между ними.

#### Повторение

Некоторые специальные символы используются для указания того, сколько раз символ будет повторяться
в тексте. Эти специальные символы — плюс `+`, звёздочка `*` и вопросительный знак `?`.

##### Звёздочка `*`

Ставим звёздочку `*` в конце, чтобы указать, что символ вообще не совпадает или может совпадать много раз.

```bash
echo "br ber beer" | rg "be*r"
```

##### Знак плюс `+`

Чтобы указать, что символ может встречаться один или несколько раз, ставим знак плюс `+` в конце.

```bash
echo "br ber beer" | rg "be+r"
```

##### Вопросительный знак `?`

Чтобы указать, что символ является необязательным, ставим в конце вопросительный знак `?`.

```bash
echo "color, colour" | rg "colou?r"
```

#### Фигурные скобки `{}`

- `{n}` — ровно n вхождений
- `{n,}` — не менее n вхождений
- `{x,y}` — от x до y вхождений

```bash
echo "ber beer beeer beeeer" | rg "be{2}r"
echo "ber beer beeer beeeer" | rg "be{3,}r"
echo "ber beer beeer beeeer" | rg "be{3,5}r"
```

#### Круглые скобки `()`

Мы можем сгруппировать выражение и использовать эти группы для ссылки или обеспечения
соблюдения некоторых правил, т.е. применяем квантификаторы к группе, а не к одному символу

```bash
# повторение группы
echo "ha haha hahaha" | rg "(ha)+"
```

##### Ссылка на группу

Слово `ha` и `haa` сгруппированы ниже. Первая группа используется путём записи `\1`,
чтобы избежать перезаписи. Здесь 1 и 2 обозначают порядок группировки.

```bash
# Флаг --pcre2 включает ссылки
echo "ha-ha,haa-haa" | rg --pcre2 "(ha)-\1,(haa)-\2"
```

##### Группа без захвата `(?:)`

Вы можете сгруппировать выражение и убедиться, что оно не перехватывается ссылками.

```bash
echo "ha-ha,haa-haa" | rg --pcre2 "(?:ha)-ha,(haa)-\1"
```

### Вертикальная черта `|`

Позволяет указать, что выражение может быть в разных частях строки. Все возможные операторы
записываются через `|` — это отличается от `[abc]`, работающего на уровне символов.

```bash
echo "cat rat dog" | rg "(c|r)at|dog"
```

### Escape-символ `\`

Специальные символы `{} [] / \ + * . $ ^ | ?` можно экранировать с помощью косой черты `\`.

```bash
echo "(*) Asterisk." | rg "(\*|\.)"
```

### Знак каретки `^`

Выбор только по началу строки.

```bash
echo "2. 1 tsp sunflower oil" | rg "^[0-9]"
```

### Знак доллара `$`

Выбор только по концу строки.

```bash
echo "https://website.com/html5-features.html" | rg "html$"
```

### Позиционные проверки (Lookarounds)

Если мы хотим, чтобы фраза находилась до или после другой фразы, используем lookaround.

#### Позитивная опережающая проверка `(?=)`

Выбирает то, после чего стоит указанное выражение.

```bash
# для lookaround нужен флаг --pcre2
# здесь хотим, чтобы PM оказалось после числа
echo "Date 4 Aug 3PM" | rg --pcre2 "\d+(?=PM)"
```

#### Негативная опережающая проверка `(?!)`

Выбирает то, после чего НЕ стоит указанное выражение.

```bash
# для lookaround нужен флаг --pcre2
echo "Date 4 Aug 3PM" | rg --pcre2 "\d+(?!PM)"
```

#### Позитивная ретроспективная проверка `(?<=)`

Выбирает то, перед чем стоит указанное выражение.

```bash
# для lookaround нужен флаг --pcre2
echo 'Product Code 1064 Price $5' | rg --pcre2 '(?<=\$)\d+'
```

#### Негативная ретроспективная проверка `(?<!)`

Выбирает то, перед чем НЕ стоит указанное выражение.

```bash
# для lookaround нужен флаг --pcre2
echo 'Product Code 1064 Price $5' | rg --pcre2 '(?<!\$)\d+'
```

### Флаги

Флаги изменяют вывод выражения, поэтому их ещё называют modifiers.

#### Глобальный флаг `/g`

Заставляет выражение выбирать все совпадения. Без него выбирается только первое.

```bash
echo 'domain.com, test.com, site.com' | rg --pcre2 '\w+\.com'
```

#### Многострочный флаг `/m`

Regex видит весь текст как одну строку, но флаг multiline обрабатывает каждую строку отдельно.

```bash
echo 'domain.com\ntest.com\nsite.com\n' | rg --pcre2 '\w+\.com'
```

#### Флаг без учёта регистра `/i`

Убирает чувствительность к регистру.

```bash
echo 'domain.com\ntest.com\nsite.com\n' | rg -i --pcre2 '\w+\.com'
```

### Жадное соответствие

Regex по умолчанию выполняет жадное сопоставление — подбор будет максимально продолжительным.

```bash
echo 'ber beer beeer beeeer' | rg --pcre2 '.*r'
```

### Ленивое соответствие

Останавливается при первом сопоставлении. Добавьте `?` после `*`.
> В ripgrep не работает (разобраться).

```bash
echo 'ber beer beeer beeeer' | rg --pcre2 '.*?r'
```

## Потоки и пайпы

### stdin/stdout/stderr
Каждый процесс при запуске получает три открытых файловых дескриптора от ядра. По умолчанию все три смотрят на терминал. Это просто числа в таблице дескрипторов процесса — за ними может стоять что угодно: терминал, файл, pipe, сокет:
- fd=0  stdin   — стандартный ввод
- fd=1  stdout  — стандартный вывод
- fd=2  stderr  — стандартный вывод ошибок

### Перенаправление и запись
```bash
# > — перенаправление stdout в файл
echo "hello" > file.txt
# открывает file.txt с флагами O_WRONLY|O_CREAT|O_TRUNC
# fd=1 процесса теперь смотрит на file.txt
# файл создаётся если нет, обрезается если есть

# >> — append
echo "hello" >> file.txt
# то же самое но O_APPEND вместо O_TRUNC
# запись всегда идёт в конец, файл не обрезается

# | — pipe между процессами
cmd1 | cmd2
# ядро создаёт буфер 64КБ в RAM
# fd=1 cmd1 → write-end буфера
# fd=0 cmd2 → read-end буфера
# данные никогда не касаются диска
```

### tee
Разветвление потока. Pipe однонаправлен: данные идут только вперёд. tee решает задачу «хочу и в файл записать, и дальше по конвейеру передать»
```bash
cmd | tee file.txt | cmd2
# внутри tee
# читает stdin
#    └─→ пишет в file.txt
#    └─→ пишет в stdout (дальше по pipe)
```

```bash
cmd1 2>/dev/null | tee raw.log | grep "OK" >> results.log 2>&1

#cmd1
#  stderr  → /dev/null          (2>/dev/null)
#  stdout  → pipe1 → tee
#                tee → raw.log  (пишет копию)
#                tee → pipe2 → grep
#                                 grep stdout → results.log  (>>)
#                                 grep stderr → results.log  (2>&1)
```

### Пайпы
Однонаправленный канал межпроцессорного вз-ия. Чаще всего используются в скриптах для связи ввода-вывода команд, но и например в go как конкурентный паттерн.

cmd1 | cmd2 | .... | cmdN


### Логика выполнения
- Конвеер обеспечивает асинхронное выполнение команд с использованием буферизации ввода/вывода. Таким образом все команды в конвейере работают параллельно, каждая в своем процессе(как и в go).

- Размер буфера начиная с ядра версии 2.6.11 составляет 65536 байт (64Кб) и равен странице памяти в более старых ядрах. При попытке чтения из пустого буфера процесс чтения блокируется до появления данных. Аналогично при попытке записи в заполненный буфер процесс записи будет заблокирован до освобождения необходимого места.

- Буфер выполняется в RAM, то есть создаём буферизированный канал в памяти ядра(go) и доступ к нему осуществляется через файловые дескрипторы. ВСПОМНИ РЕАЛИЗАЦИЮ КОНВЕЙЕРА С ВОЗВРАЩЕНИЕМ КАНАЛА ИЗ ФУНКЦИИ.

```c
// структура буффера
struct pipe_inode_info {
    unsigned int head;        /* индекс записи  */
    unsigned int tail;        /* индекс чтения  */
    unsigned int max_usage;   /* 16 страниц     */
    struct pipe_buffer bufs[PIPE_DEF_BUFFERS];  /* 16 слотов */
};
```

- pipe(2) создаёт буфер в ядре и возвращает пару fd (read-end + write-end)
- dup2(2) переназначает эти fd на стандартные потоки (0, 1, 2) дочерних процессов
- read(2) / write(2) работают через эти fd — процессы не знают и не заботятся, что за ними стоит: файл на диске, терминал или pipe-буфер в RAM

Именно в этой абстракции файлового дескриптора и есть элегантность Unix: один и тот же интерфейс, разная реализация под капотом.

```bash
# 1. bash вызывает pipe(2) дважды, то есть ядро создаёт два анонимных буфера в памяти. Никаких файлов на диске не возникает.

# 2. Раздача дескрипторов дочерним процессам.
# Для cat:
fork() → child1
  dup2(fd1[1], STDOUT_FILENO)  # stdout cat → запись в pipe1
  close(fd1[0]); close(fd2[0]); close(fd2[1])

# Для grep:
fork() → child2
  dup2(fd1[0], STDIN_FILENO)   # stdin grep ← чтение из pipe1
  dup2(fd2[1], STDOUT_FILENO)  # stdout grep → запись в pipe2
  close(fd1[1]); close(fd2[0])

# Для wc:
fork() → child3
  dup2(fd2[0], STDIN_FILENO)   # stdin wc ← чтение из pipe2
  close(fd2[1]); close(fd1[0]); close(fd1[1])
```

### Порядок перенаправления вывода-ошибок
- 2>&1 - перенаправление fd=2(stderr) туда же, куда и fd=1(stdout)
```bash
cmd > file.txt 2>&1
# stdout → file.txt
# stderr → туда же, куда stdout → file.txt
```

- 2>&1 >/dev/null - перенаправляет в dev/null только стандартный вывод


- >/dev/null 2>&1 - перенаправляет и stdout  stderr в dev/null


- 2>/dev/null - направление ошибки в dev/null

### Практика
```bash
2>&1 | — stderr в pipe
# только ошибки идут в grep, stdout подавлен
ls ./Documents ./нет_файла 2>&1 >/dev/null | grep "No such"
ls: cannot access './нет_файла': No such file or directory

# |& — короткая запись того же самого
ls ./Documents ./нет_файла |& grep "No such"
ls: cannot access './нет_файла': No such file or directory
```

```bash
# pipefail — код завершения
# без pipefail — всегда 0 (код wc), даже если ls упал
ls ./нет_файла 2>/dev/null | wc -l
echo $?   # → 0

# с pipefail — возвращает код первой упавшей команды
set -o pipefail
ls ./нет_файла 2>/dev/null | wc -l
echo $?   # → 2

# ловушка: grep без совпадений → код 1
set -o pipefail
echo "hello" | grep "xyz" | wc -l
echo $?   # → 1, хотя wc отработал нормально

# обход: || true гасит ненулевой код grep
echo "hello" | grep "xyz" || true | wc -l
echo $?   # → 0
```
