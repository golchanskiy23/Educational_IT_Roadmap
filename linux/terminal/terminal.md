# Linux CLI

## Навигация и файлы

### ls
ls [опции] [директория]

Выдача информации о файлах или каталогах. Фвйлы -> каталоги в алфавитном порядке
- ls -a - вывод всех файлов, в том числе скрытых
- ls -l - вывод в длинном формате, включая режим доступа, владельца, группу, время модификации, размер файлов в байтах, число ссылок на файл

```bash
ls -l /util/by
# ugo - тип файла, user, group, other, итого 9 бит на доступ
-rwxr-xr-x   1 root     sys           50 Jun 22 10:42 /util/by
```

r - чтение, w - запись, x - запуск, '-' - отсутствие доступа

### find
find [директория] [опции] [действие] 

Поиск файлов по любому параметру: имя, содержимое, дата добавления и т.д. Шаг за шагом проходит через всю иерархию директорий.

```bash
$ find . -name ".txt"
./s.txt

$ find . -iname "s.txt"
./s.txt
./S.txt

$ find . -name "*.png"
./babutafb.png
./babutafacebook.png
./Moodle2.png
./moodle.png
./moodle/moodle1.png
./genxfacebook.png

```

- 2>/dev/null - перенаправляет сообшения об ошибках по сути в никуда

Поиск по типу файла:
- f – простые файлы;
- d – каталоги;
- l – символические ссылки;
- b – блочные устройства (dev);
- c – символьные устройства (dev);
- p – именованные каналы;
- s – сокеты;

```bash
# по размеру
$ find . -size +1G
./Microsoft_Office_16.29.19090802_Installer.pkg
./android-studio-ide-183.5692245-mac.dmg
```

```bash
# atime - поиск файлов по времени последнего доступа 
# здесь ищет файлы к которым не обращались последние 180 дней
$ find . -atime +180
```

К команде find можно добавить действия, которые будут произведены с результатами поиска:
- -delete — Удаляет соответствующие результатам поиска файлы
- -ls — Вывод более подробных результатов поиска с: размерами файлов, количеством inode.
- -print Стоит по умолчанию, если не указать другое действие. Показывает полный путь к найденным файлам.
- -exec Выполняет указанную команду в каждой строке результатов поиска.

```bash
# поисковые параметры можно соединять и выводить
$ find . -type f -and -user bloom -and -size +500M -and -atime +10 -exec ls -lah {} \;
```

### locate
locate [опции] [шаблон_для_поиска]

Используется для поиска файлов, расположенных на машине пользователя или на сервере, однако ведёт поиск в собственной базе данных, обновляемой через cron командой updatedb. Но из-за нечастого обновления данные могут быть неактуальными. Более скудный набор фильров (только по имени и пути).

```bash
$ locate filename        # найти файл по имени
$ locate -i filename     # без учёта регистра
$ locate -c filename     # только количество совпадений
$ locate -n 5 filename   # показать первые 5 результатов
$ locate -r '\.mkv$'     # поиск по регулярному выражению
$ sudo updatedb          # обновить базу данных вручную
```

### tree
tree [опции]

Изображает структуру каталогов в виде дерева, причём имеет широкую почву для фильтрации параметров и форматирования вывода.

```bash
# выводит только директории(d), включая скрытые(a) и время их последнего изменения(D), v(упорядочивание по вложенности директорий) + существует ограничение на число файлов, причём меньше 25 не установить
$ tree -avdD --filelimit 26
```

Опять же куча параметров -> в man.

## Работа с текстом

### cat
cat [опции] [file1] [file2] [...] 

Используется для чтения, создания и объединения файлов.

```bash
# стандартный поток ввода обозначается как '-', т.о. комбирируем ввод и вывод и нумеруем все строки(n)
$ cat -n file1 - file2 
# нумеруем только непустые строки(b)
$ cat -b file 
```

При отсутствии переданных файлов cat будет читать данные из стандартного ввода(на экран). Также можно записывать данные из стандартного потока в файл.
```bash
# здесь выводим содердимое двух файлов и записываем в третий
$ cat file1 file2 > file3
```

### less
less [опции] [файл]

Для просмотра содержимого файла и вывода небольшими фрагментами. Зачастую используется для просмотра больших файлов и листать в обоих направлениях. 

При открытии файла открывается командная строка, в которой можно искать вниз/вверх по файлу -> (/error) / (?error). Также там работают регулярки.

#### Некоторые команды:
```bash
# нумерация строк файла
$ less -N /var/log/syslog
# открытие файла с текущей позиции
$ less +100 bigfile.txt
# ВАЖНО: переход в режим реального времени - новые строки появляются автоматически
$ less +F /var/log/nginx/access.log
```

#### Для нескольких файлов:
```bash
# с помощью командной строки переключаться между файлами
# :n - следующий, :p - предыдущий, :d - удаление текущего из списка
$ less file1.log file2.log file3.log
```

#### Пайпы
less также позволяет выводить вывод других команд через конвейер

```bash
$ dmesg | less
```

#### Сравнение инструментов

| Инструмент | Прокрутка назад | Поиск         | Загрузка файла | Слежение |
|------------|-----------------|---------------|----------------|----------|
| `cat`      | —               | —             | Целиком        | —        |
| `more`     | —               | Только вниз   | Целиком        | —        |
| `less`     | Да              | В обе стороны | Кусками        | `+F`     |

#### Шпаргалка less

| Задача                  | Команда / клавиша  |
|-------------------------|--------------------|
| Открыть файл            | `less filename`    |
| Открыть с позиции конца | `less +G filename` |
| Слежение за файлом      | `less +F filename` |
| Номера строк            | `less -N filename` |
| Сжать пустые строки     | `less -s filename` |
| Игнорировать регистр    | `less -i filename` |
| Поиск вниз              | `/pattern`         |
| Поиск вверх             | `?pattern`         |
| Следующее совпадение    | `n`                |
| Предыдущее совпадение   | `N`                |
| В начало файла          | `g`                |
| В конец файла           | `G`                |
| Следующий файл          | `:n`               |
| Выйти                   | `q`                |


### head
head [опции] [файл1] [файл2] [...]

Вывод начальных строк(по умолчанию 10) для каждого файла.

```bash
# вывод первых 15 строк
$ head -n 15 file1.txt
# вывод первых 500 байт
$ head -c 500 file1.txt
```

- аналогично для tail

### tail -f
Выполняет ту же функцию, что и less +F, то есть следит за файлами в реальном времени.

```bash
# начать с последних 50 строк и следить
$ tail -n 50 -f /var/log/nginx/access.log
# следить за несколькими файлами сразу
$ tail -f /var/log/nginx/access.log /var/log/nginx/auth.log
```

Из минусов - отсутствие переключения между выводом файла и режимом слежки (в less это доступно).

### sort
sort [опции] [file1] [file2] [...]

Утилита для вывода текстовых строк в определенном порядке. Проще говоря, для сортировки. Опции определяют критерий сортировки.

```bash
# сортировка по 2-ой колонке по числам
$ ls -l | sort -nk2
# можно также задавать приоритеты сортировки
$ ls -l | sort -nk5,5 -k9
```

### uniq
uniq [опции] [файл для чтения] [файл для записи]


Позволяет работать с повторяющимися строками, допустим выводить только их или удалить их из файла.
```bash
# сравнение по первым пяти символам
$ echo -e небо исполосовано молниями\\nоблака на небе\\nоблака разогнал ветер\\nоблака закрыли солнце\\nсолнце светит ярко\\nзвезды кажутся огромными | uniq -w5
# убирает из вывода все дубликаты
$ echo -e небо\\nоблака\\nоблака\\nоблака\\nсолнце\\nзвезды | uniq
# выводит только уникальные строки
$ echo -e небо\\nоблака\\nоблака\\nоблака\\nсолнце\\nзвезды | uniq -u
```

### cut
cut [опция] [файл]

Используется для вырезания части текста.
Можно вырезать как символы, так и байты, причём как отдельно-взятые, так и диапазон.

```bash
# также можно вырезать первые элементы по столбцам. По разделителю ':'
$ cut -d ':' -f 1 cut_command.txt
# здесь результат - первые 3 слова
$ echo "He was extraordinarily particular about politeness in others." | cut -d ' ' -f 1,2,3
# по аналогии с срезами go можем выделять диапазоны вырезаемых элементов
# здесь первые 2
$ echo "We became still more friendly." | cut -d ' ' -f -2
```
